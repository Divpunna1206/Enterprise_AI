Act as a senior Product Designer, UI Engineer, and Figma expert with 20+ years of experience. Design a complete high-fidelity Figma prototype for a multi-tenant SaaS LMS platform named “Ascentia” / “Enterprise AI Learning SaaS”.

The platform is built for schools, coaching institutes, and learning organizations. The main product flow must follow:

Org Admin → Teacher → Students → Parents

Design the product as a modern SaaS LMS with AI-powered learning, classroom management, assignments, analytics, student progress tracking, parent visibility, and institute-level control.

The final Figma file should include:

1. Web dashboard prototype
2. Mobile app screens for student and parent
3. Landing page
4. Design system
5. Clickable prototype flow
6. Responsive layout guidance

Brand Direction:
Create a premium, clean, modern education SaaS interface. The UI should feel trustworthy, enterprise-ready, simple, and scalable. Use a white and blue theme with soft gradients, rounded cards, glassmorphism where appropriate, clean spacing, and modern dashboard layouts.

Suggested visual style:

* Primary color: Deep Blue / Royal Blue
* Secondary: Sky Blue / Cyan
* Background: White / Soft Blue / Light Gray
* Accent: Green for success, Orange for warnings, Red for alerts
* Typography: Inter, Manrope, or similar modern sans-serif
* Components: Rounded cards, clean tables, soft shadows, progress charts, role-based navigation
* Style inspiration: Linear, Stripe, Google Classroom, Duolingo dashboard, modern LMS SaaS platforms

Core Product Concept:
This is a SaaS LMS where an organization admin manages the entire institute. Teachers manage classes, lessons, assignments, student performance, and AI learning support. Students access courses, AI tutor, lessons, quizzes, tests, progress, and feedback. Parents view their child’s attendance, performance, assignments, fees, reports, and teacher updates.

Important User Roles:

1. Org Admin
   The Org Admin controls the school/coaching institute account. They can manage teachers, students, parents, classes, subjects, batches, subscriptions, reports, announcements, branding, and overall platform usage.

2. Teacher
   Teachers manage assigned classes, subjects, lessons, homework, tests, attendance, student progress, AI-generated quizzes, doubt-solving reports, and parent communication.

3. Student
   Students access their dashboard, classes, lessons, AI tutor, assignments, quizzes, mock tests, learning progress, weak topics, badges, timetable, and teacher feedback.

4. Parent
   Parents monitor the child’s learning journey, attendance, performance, assignments, test scores, teacher remarks, fee status, announcements, and progress reports.

Prototype Flow Requirements:

Create clickable flows for:

Flow 1: Org Admin Onboarding
Login → Organization Dashboard → Add Teacher → Create Class → Add Students → Link Parents → Assign Teacher → Publish Course

Flow 2: Teacher Daily Workflow
Login → Teacher Dashboard → Select Class → Mark Attendance → Create Assignment → Review Student Submission → Send Parent Update

Flow 3: Student Learning Workflow
Login → Student Home → Open Course → Watch Lesson → Ask AI Tutor → Complete Quiz → View Progress

Flow 4: Parent Monitoring Workflow
Login → Parent Home → View Child Progress → Check Attendance → View Test Result → Read Teacher Remark → Message Teacher

Flow 5: AI Learning Workflow
Student asks doubt → AI Tutor responds → Doubt saved → Teacher sees AI Tutor Log → Teacher sends clarification → Parent sees progress update
